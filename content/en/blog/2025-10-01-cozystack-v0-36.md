### 😜 Cozystack v0.36: серверное шифрование для S3, монитор состояния кластера Kube-OVN, документация REST API

Новая версия Cozystack сосредоточена на стабильности, наблюдаемости и гибкой настройке управляемых приложений.

![](https://cdn-images-1.medium.com/max/800/1*nSnbuqjkZ66y1L8T6tEmEw.png)

### 👉 Основные возможности и улучшения

#### Лимиты ресурсов на пространство имён для арендаторов

Управление ресурсами для арендаторов Cozystack получило финальный патч и теперь переведено в статус стабильной функции. Администраторы платформы могут задавать явные лимиты CPU, памяти и хранилища для пространства имён каждого арендатора через спецификацию арендатора. Это не позволяет ни одному арендатору потреблять больше своей доли ресурсов кластера, обеспечивая стабильность кластера и гарантированный уровень обслуживания для каждого арендатора.

#### Монитор состояния кластера Kube-OVN

Новый компонент под названием Kube-OVN Plunger непрерывно отслеживает состояние центрального управляющего кластера сети Kube-OVN. Этот внешний агент собирает информацию о статусе и консенсусе кластера OVN, предоставляя метрики Prometheus и потоковую передачу событий в реальном времени через SSE. В результате это обеспечивает намного лучшую наблюдаемость уровня виртуальной сети и помогает поддерживать надёжную и наблюдаемую сеть в Cozystack. Это изменение открывает путь к автоматизированным операциям с базой данных Kube-OVN и восстановлению в отдельных нестандартных случаях.

#### Настраиваемый аддон CoreDNS для Kubernetes

Cozystack представляет специальный аддон CoreDNS для более гибкого управления DNS кластера. CoreDNS теперь развёртывается через Helm-чарт и может настраиваться с помощью пользовательских значений в спецификации кластера, включая автомасштабирование, число реплик и изменение service IP. CoreDNS теперь можно настраивать в панели управления и с помощью Cozystack API.

#### Детальная настройка сервиса SeaweedFS

Сервис хранилища SeaweedFS S3 в Cozystack теперь гораздо гибче настраивается на уровне компонентов. Helm-чарт для SeaweedFS теперь включает независимую конфигурацию для каждого компонента и его ресурсов. Сюда входят master-узлы, серверы томов с поддержкой нескольких зон, filer'ы, обеспечивающая база данных и шлюз S3. Администраторы могут задавать параметры для каждого компонента, такие как число реплик, доступные CPU, память и размер хранилища.

#### Серверное шифрование для S3

Cozystack v0.36.0 включает SeaweedFS 3.97, добавляя поддержку серверного шифрования S3-бакетов (SSE-C, SSE-KMS и SSE-S3).  
Критическое изменение: при обновлении Cozystack SeaweedFS будет обновлён до более новой версии, а спецификация сервисов будет преобразована в новый формат.

#### Пользовательские профили ресурсов для Ingress-контроллера

Контроллер NGINX теперь настраивается на уровне отдельных реплик. Настройки включают requests/limits для CPU и памяти подов ingress-контроллера — либо прямыми значениями, либо с помощью одного из доступных пресетов.

#### Встроенное обнаружение соседей на базе LLDP в Talos

Cozystack теперь включает расширение LLDPD в свой образ Talos OS, обеспечивая поддержку Link Layer Discovery Protocol (LLDP) «из коробки». Это означает, что каждый узел может автоматически обнаруживать и анонсировать своих сетевых соседей и топологию без какой-либо ручной настройки.

#### Использование внешнего IP для исходящего трафика в виртуальных машинах

Когда виртуальной машине назначен внешний IP, она теперь всегда будет использовать его для исходящего трафика, независимо от используемого внешнего метода.

### 🔔 Новые версии компонентов

\- Обновление LINSTOR до v1.31.3  
- Обновление SeaweedFS до v3.97   
- Обновление Kube-OVN до 1.14.5  
- Замена образов Bitnami альтернативами во всех чартах
