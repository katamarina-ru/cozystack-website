### Cozystack v0.41: MongoDB, кнопка «Редактировать» в панели управления, интерфейс квот ресурсов, безопасность JWT и Gateway API в cert-manager

Версия 0.41 расширяет каталог баз данных за счёт MongoDB и значительно улучшает панель управления и уровень безопасности.

### Основные возможности и улучшения

#### MongoDB как управляемое приложение

**MongoDB** пополняет каталог приложений Cozystack как полностью управляемый сервис со следующими возможностями:

- Развёртывание набора реплик (replica set) с настраиваемым числом реплик.
- Постоянное хранилище с настраиваемым размером.
- Управление ресурсами (requests/limits для CPU и памяти).
- Встроенная интеграция с мониторингом.

#### Кнопка «Редактировать» в панели управления

У всех ресурсов в панели управления теперь есть **кнопка «Редактировать»**, что позволяет пользователям изменять любое развёрнутое приложение прямо из интерфейса, не переключаясь на CLI.

#### Использование квот ресурсов в панели управления

Страница сведений об арендаторе теперь отображает **использование квот ресурсов**, давая арендаторам и операторам ясную картину того, какая часть выделенных им ресурсов израсходована.

#### Проверка JWT-токенов

Панель управления теперь выполняет **проверку JWT-токенов**, закрывая брешь в безопасности и гарантируя, что только аутентифицированные пользователи могут взаимодействовать с API платформы.

#### Поддержка Gateway API в cert-manager

cert-manager теперь **автоматически включает поддержку Gateway API**, когда активен аддон Gateway API, упрощая управление TLS-сертификатами для ingress на основе шлюза.

#### Улучшения гибкости ВМ

- ВМ теперь могут **изменять поле `external`** после создания.
- ВМ могут **переключаться между instancetype и пользовательскими ресурсами** после создания.

### Исправления (v0.41.1–v0.41.11)

Ключевые исправления в 11 патч-релизах включают:
- Валидация enum для exposeMethod в IngressNginx.
- Для VMAgent значение VPA minReplicas установлено в 1.
- Отображение ресурсов Service и Ingress для приложения Kubernetes в панели управления.
- Разделены значения брендинга Keycloak.
- Исправлен serviceaccount для CoreDNS в соответствии с RBAC при начальной загрузке (bootstrap) Kubernetes.
- Добавлен startupProbe для панели управления, чтобы предотвратить перезапуски на медленном оборудовании.
- Запрещено удаление resourcequotas для администратора арендатора.
- Kube-OVN обновлён до v1.15.3.
