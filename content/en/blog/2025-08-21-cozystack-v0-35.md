### Cozystack v0.35: внешние источники приложений, выделенные кластеры S3 и мониторинг, поддержка Hetzner RobotLB

Новая версия Cozystack делает значительный шаг вперёд в развитии своей модульной (или: декомпозированной) архитектуры, позволяя пользователям быстро интегрировать собственные приложения и сервисы. Это существенно расширяет доступную «из коробки» функциональность платформы под конкретные бизнес-задачи. И это ещё не всё!

![](https://cdn-images-1.medium.com/max/800/0*BTfwy72MMG2NBvvm)

> Что такое Cozystack?

> Cozystack — это бесплатный PaaS и фреймворк для построения облаков, объединяющий виртуальные машины, контейнеры и рабочие нагрузки GPU под управлением Kubernetes. Компании могут превратить оборудование в облако: предлагать пользователям или клиентам управляемый K8s, виртуальные машины, управляемые базы данных, приложения и GPU-сервисы. Благодаря интеграции с KubeVirt, мультиарендности (multi-tenancy) и простоте bare-metal она позволяет предприятиям развёртывать ИИ, базы данных или периферийные (edge) приложения без привязки к вендору. Cozystack — проект CNCF Sandbox.

### Основные возможности и улучшения

#### Внешние источники приложений в Cozystack

Теперь Cozystack поддерживает [добавление внешних пакетов приложений](https://cozystack.io/docs/applications/external/) в каталог приложений платформы. Администраторы платформы могут включать собственные или сторонние приложения наряду со встроенными, используя API Cozystack.

Для добавления приложения нужно создать пакет приложения, аналогичный тем, что входят в состав Cozystack в [packages/apps](https://github.com/cozystack/cozystack/tree/main/packages/apps). Использование внешних пакетов обеспечивается новым CustomResourceDefinition (CRD) под названием CozystackResourceDefinition и соответствующим контроллером (reconciler), который отслеживает эти ресурсы.

Добавьте собственное управляемое приложение, воспользовавшись [документацией](https://cozystack.io/docs/applications/external/) и примером на [github.com/cozystack/external-apps-example](https://github.com/cozystack/external-apps-example).

#### Улучшения API Cozystack

Этот релиз приносит значительные улучшения в спецификации OpenAPI для всех управляемых приложений в Cozystack, включая базы данных, Kubernetes арендаторов, виртуальные машины, мониторинг и другие. Эти изменения включают более точные определения типов для полей, которые раньше описывались лишь как обобщённые объекты, и теперь многие поля имеют ограничения на значения. Теперь многие возможные ошибки конфигурации обнаруживаются сразу же при запросе к API, а не позднее — при неудачном развёртывании.

Теперь API Cozystack также отображает значения по умолчанию для ресурсов приложений. Большинство остальных полей теперь имеют разумные значения по умолчанию, когда такие значения возможны.

Все эти изменения прокладывают путь к новому UI Cozystack, который сейчас находится в разработке.

#### Поддержка Hetzner RobotLB

MetalLB — балансировщик нагрузки по умолчанию, входящий в состав Cozystack, — создан для bare metal и самостоятельно размещаемых виртуальных машин, но не поддерживается большинством облачных провайдеров. Например, Hetzner предоставляет собственный сервис RobotLB, который Cozystack теперь поддерживает как опциональный компонент.

Прочитайте обновлённое руководство по [развёртыванию Cozystack на Hetzner.com](https://cozystack.io/docs/install/providers/hetzner/), чтобы узнать больше и развернуть собственный кластер Cozystack на Hetzner.

#### Сервис S3: выделенные кластеры и мониторинг

Теперь можно развёртывать выделенные кластеры Cozystack для запуска сервиса S3 на базе SeaweedFS. Благодаря поддержке [интеграции с удалёнными эндпоинтами filer](https://cozystack.io/docs/operations/stretched/seaweedfs-multidc/) вы можете подключить основной кластер Cozystack к использованию хранилища S3 в выделенном кластере.

В целях безопасности администраторы платформы теперь могут настроить в приложении SeaweedFS список IP-адресов или диапазонов CIDR, которым разрешён доступ к сервису filer.

SeaweedFS также был интегрирован в стек мониторинга и теперь имеет собственную панель управления Grafana. Вместе эти улучшения помогают пользователям Cozystack строить более надёжный, масштабируемый и наблюдаемый сервис S3.

#### ClickHouse Keeper

Приложение ClickHouse теперь включает сервис ClickHouse Keeper для повышения надёжности и доступности кластера. Этот компонент по умолчанию развёртывается с каждым кластером ClickHouse.

Узнайте больше в [справочнике по конфигурации ClickHouse](https://cozystack.io/docs/applications/clickhouse/#clickhouse-keeper-parameters).
