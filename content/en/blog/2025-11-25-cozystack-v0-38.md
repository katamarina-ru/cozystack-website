### Cozystack v0.38: виртуальное частное облако, VNC-консоль, настраиваемые версии Kubernetes для worker-узлов и принудительное использование HTTPS

Версия 0.38 приносит возможности сетевой изоляции, улучшенный доступ к виртуальным машинам и усиление безопасности на всей платформе.

### Основные возможности и улучшения

#### Виртуальное частное облако (VPC)

Ключевая возможность v0.38 — **поддержка VPC** с интеграцией Multus CNI. Операторы теперь могут создавать изолированные виртуальные сети с:

- **Управлением подсетями** для гибкой организации сети.
- **Сетевой изоляцией** между арендаторами на сетевом уровне.
- Полной интеграцией с панелью управления Cozystack для управления жизненным циклом VPC.

#### VNC-консоль для виртуальных машин

У виртуальных машин теперь есть **VNC-консоль**, доступная прямо из панели управления, что обеспечивает графический доступ к виртуальным машинам без внешних инструментов.

#### Настраиваемые версии Kubernetes для worker-узлов

Операторы теперь могут независимо настраивать **версии Kubernetes для worker-узлов** в кластерах арендаторов, что позволяет проводить постепенные обновления и закреплять версии.

#### Усиление безопасности

- **Принудительное использование только HTTPS** для API Cozystack.
- Закрыт внешний доступ к портам Flux Operator.
- Обновления образов Redis с исправлениями безопасности.

### Обновления компонентов

- LINSTOR v1.32.3
- Talos Linux v1.11.3
- Kube-OVN v1.14.11
- Piraeus Operator v2.10.1
- MariaDB Operator обновлён

### Исправления (v0.38.1–v0.38.8)

Ключевые исправления в 8 патч-релизах:
- Увеличен срок действия CA-сертификата SeaweedFS.
- Метки приложения CoreDNS приведены в соответствие со значениями Talos по умолчанию.
- Обновлён piraeus-operator до v2.10.2 для надёжных проверок fsck.
- Поддержка планирования через nodeAffinity для виртуальных машин Windows.
- SeaweedFS обновлён до v4.02.
- Удалён лимит памяти для Multus из-за непредсказуемых всплесков потребления памяти при запуске.
